Naming convention of pyd files:

_ANSI: these are the Dragon pre 12 versions for python 2.6, 2.6 and 2.7 (Rudiger Wilke)
       these are not used any more with python 3 and are removed.

_UNICODE: these are the Dragon 12, 13, 14 versions for python 2.6 and 2.7 (Rudiger)
       these are not used any more with python 3 and are removed.

3.x_UNICODE: these are introduced, in order to support Natlink with python3 for versions 12, 13 and 14 of Dragon.     

_2.7_Ver15: Dragon 15 version for python 2.7 (Rudiger)
       this one is not used any more with python 3 and is removed.


natlink2_2.7_UNICODE: pyd for Dragon 12, 13, 14 and python 2.7 as compiled (Jan Scheffczyk)(knork)

now: 3.x gives the python version, mostly python38-32 at the moment (installed for all users)

_UNICODE is trying to support versions 12, 13 and 14 of Dragon

_Ver15 is for Dragon 15.
